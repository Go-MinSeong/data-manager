# S3 Manager 설치 가이드 (받는 사람용)

AWS S3에서 파일을 받고/올리는 macOS 메뉴바 앱입니다. 각자 자기 Mac에서, 자기 AWS 자격증명으로 사용합니다.

## 0. 요구사항
- **macOS (Apple Silicon — M1 이상)**. Intel Mac은 별도 빌드가 필요하니 배포자에게 문의하세요.
- 본인의 **AWS Access Key / Secret Key** (또는 이미 설정된 `~/.aws/credentials` 프로파일).

## 1. 설치
1. 전달받은 **`S3-Manager-arm64.zip`** 을 더블클릭해 압축을 풉니다.
2. 나온 **`S3 Manager.app`** 을 **`응용 프로그램(Applications)`** 폴더로 드래그합니다.

## 2. 첫 실행 (중요 — Gatekeeper)
이 앱은 Apple 공증(notarize)을 받지 않아서, 처음엔 macOS가 실행을 막습니다. **한 번만** 아래로 열어주면 그 다음부턴 그냥 열립니다.

**방법 A (권장):**
1. `응용 프로그램`에서 **`S3 Manager.app`을 우클릭 → "열기"**.
2. 경고창이 뜨면 다시 **"열기"** 를 클릭.

**방법 B (방법 A에서 "손상되었습니다"라고 나올 때):** 터미널에서 한 줄 실행 후 다시 실행
```bash
xattr -dr com.apple.quarantine "/Applications/S3 Manager.app"
```

> 정상 실행되면 **Dock에는 안 뜨고 상단 메뉴바에 구름(S3) 아이콘**이 생깁니다.

## 3. 사용
1. 메뉴바의 **구름(S3) 아이콘 클릭 → "창 열기"**.
2. **연결** 화면에서 자격증명 선택:
   - **직접 입력**: Access Key / Secret Key / 리전 입력 (저장 안 됨, 메모리에만).
   - **Keychain 저장**: "새 프로파일 저장"으로 macOS Keychain에 안전 보관 → 다음부턴 선택만.
   - **AWS 프로파일**: `~/.aws/credentials`에 프로파일이 있으면 자동으로 목록에 표시됩니다.
3. 연결되면 왼쪽 트리에서 버킷/폴더를 펼쳐 다운로드 대상을 체크 → **다운로드** 탭에서 저장 경로 선택 → 시작.

## 4. (선택) 로그인 시 자동 실행
메뉴바에 항상 떠 있게 하려면:
- **시스템 설정 → 일반 → 로그인 항목** → `+` → `S3 Manager.app` 추가.

## 5. 안전 / 보안
- 🛡️ **S3 객체 삭제 기능이 없습니다.** 읽기(목록/다운로드)와 업로드만 가능 — 실수로 S3 데이터를 지울 수 없습니다.
- 🔒 앱 서버는 **본인 Mac 내부(127.0.0.1)에서만** 동작하고, 실행마다 발급되는 토큰으로 보호돼 **다른 사람·다른 프로그램이 접근할 수 없습니다.**
- 🔑 AWS 자격증명은 **각자 본인 것**을 사용하세요. 평문으로 디스크에 저장되지 않으며, Keychain 저장 시 macOS가 암호로 보호합니다.
- 자격증명은 본인이 접근 권한을 가진 버킷에만 쓰세요.

## 6. 문제 해결
- **"열 수 없습니다 / 확인되지 않은 개발자"** → 위 2번(우클릭 → 열기).
- **"손상되었습니다"** → 위 2번 방법 B의 `xattr` 명령.
- **메뉴바에 아이콘이 안 보임** → 메뉴바가 꽉 찼을 수 있음. 다른 메뉴바 앱을 줄이거나 `⌘`드래그로 정리.
- **창이 안 뜸** → 메뉴바 아이콘 → "창 열기" 다시 클릭.
- **연결 실패** → Key/Secret/리전 확인, 해당 IAM 사용자에 `s3:ListBucket`·`s3:GetObject` 권한이 있는지 확인.
